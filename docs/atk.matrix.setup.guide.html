📁 guide d'installation - atomeek datamatrix

Configuration complète Angular 20 + PHP 8.3 + Docker sur Windows 11
📅 Date : 7 août 2025
👤 Projet : atomeek-datamatrix
🏗️ Stack : Angular 20 + PHP 8.3 + Docker
📊 Status : ✅ Prêt pour installation

1️⃣prérequis système

🖥️ système windows 11

  • Windows 11 Pro/Home (build 22000+)
  • RAM : Minimum 8GB (16GB recommandé)
  • Stockage : 20GB libres minimum
  • Virtualisation : Hyper-V activé

🛠️ outils de développement

  • Node.js : Version 20.x LTS
  • Git : Version récente
  • VS Code : Avec extensions Docker
  • Docker Desktop : Version récente

2️⃣installation docker windows 11

⚠️ Attention : Installation Docker sur Windows 11 nécessite une configuration système précise. Suivre chaque étape scrupuleusement.

🔧 étape 1 : activation de la virtualisation

Vérification BIOS/UEFI :
  1. Redémarrer le PC et accéder au BIOS/UEFI (F2, F12, DEL selon constructeur)
  2. Chercher "Virtualization Technology" ou "Intel VT-x/AMD-V"
  3. Activer la virtualisation
  4. Sauvegarder et redémarrer

🔧 étape 2 : activation wsl2 et hyper-v

PowerShell en tant qu'administrateur :
# Enable WSL2
dism.exe /online /enable-feature /featurename:Microsoft-Windows-Subsystem-Linux /all /norestart

# Enable Virtual Machine Platform
dism.exe /online /enable-feature /featurename:VirtualMachinePlatform /all /norestart

# Enable Hyper-V
dism.exe /online /enable-feature /featurename:Microsoft-Hyper-V-All /all /norestart

# Restart required
Restart-Computer

🔧 étape 3 : installation wsl2 kernel

Après redémarrage :
  1. Télécharger le kernel WSL2 : https://aka.ms/wsl2kernel
  2. Installer le fichier MSI téléchargé
  3. Définir WSL2 par défaut :
wsl --set-default-version 2

🔧 étape 4 : installation docker desktop

  1. Télécharger Docker Desktop : https://www.docker.com/products/docker-desktop
  2. Lancer l'installateur en tant qu'administrateur
  3. Cocher "Use WSL 2 instead of Hyper-V"
  4. Cocher "Add shortcut to desktop"
  5. Installer et redémarrer

🔧 étape 5 : configuration docker desktop

  1. Lancer Docker Desktop
  2. Aller dans Settings → General
  3. Vérifier "Use WSL 2 based engine"
  4. Aller dans Resources → WSL Integration
  5. Activer l'intégration avec les distributions WSL
  6. Apply & Restart

✅ vérification installation

# Test Docker
docker --version
docker run hello-world

# Test Docker Compose
docker-compose --version

🌐 configuration nginx

Chemin : /docker/nginx/nginx.conf
server {
    listen 80;
    server_name localhost;

    root /var/www/html/public;
    index index.php index.html;

    # Logs
    access_log /var/log/nginx/access.log;
    error_log /var/log/nginx/error.log;

    # Security
    server_tokens off;

    # Static files
    location ~* \.(js|css|png|jpg|jpeg|gif|ico|svg|woff|woff2|ttf|eot)$ {
        expires 1y;
        add_header Cache-Control "public, immutable";
        add_header Access-Control-Allow-Origin "*";
        access_log off;
    }

    # Main routes and API
    location / {
        try_files $uri $uri/ /index.php?$query_string;

        # CORS headers
        add_header Access-Control-Allow-Origin "*" always;
        add_header Access-Control-Allow-Methods "GET, POST, PUT, DELETE, OPTIONS, PATCH" always;
        add_header Access-Control-Allow-Headers "Origin, X-Requested-With, Content-Type, Accept, Authorization, Cache-Control" always;
        add_header Access-Control-Max-Age "3600" always;

        # Handle OPTIONS requests
        if ($request_method = OPTIONS) {
            add_header Access-Control-Allow-Origin "*";
            add_header Access-Control-Allow-Methods "GET, POST, PUT, DELETE, OPTIONS, PATCH";
            add_header Access-Control-Allow-Headers "Origin, X-Requested-With, Content-Type, Accept, Authorization, Cache-Control";
            add_header Access-Control-Max-Age "3600";
            return 200;
        }
    }

    # Uploads handling
    location /uploads/ {
        alias /var/www/html/public/uploads/;
        add_header Access-Control-Allow-Origin "*";
        expires 30d;
    }

    # PHP processing
    location ~ \.php$ {
        include fastcgi_params;
        fastcgi_pass php:9000;
        fastcgi_index index.php;
        fastcgi_param SCRIPT_FILENAME $document_root$fastcgi_script_name;
        fastcgi_param PATH_INFO $fastcgi_path_info;

        # Timeouts for large operations
        fastcgi_read_timeout 300;
        fastcgi_send_timeout 300;
        fastcgi_connect_timeout 300;

        # Buffers
        fastcgi_buffer_size 128k;
        fastcgi_buffers 4 256k;
        fastcgi_busy_buffers_size 256k;
    }

    # Security - block sensitive files
    location ~ /\. {
        deny all;
        access_log off;
        log_not_found off;
    }

    location ~ \.(env|ini|log|sh|sql)$ {
        deny all;
        access_log off;
        log_not_found off;
    }

    # Size limits
    client_max_body_size 100M;
    client_body_timeout 120s;
    client_header_timeout 120s;
}

🌐 point d'entrée api

Chemin : /atkm_back/public/index.php
<?php
/**
 * Atomeek DataMatrix API Entry Point
 * Main entry point for all API requests
 */

// CORS headers for all requests
header('Access-Control-Allow-Origin: *');
header('Access-Control-Allow-Methods: GET, POST, PUT, DELETE, OPTIONS, PATCH');
header('Access-Control-Allow-Headers: Origin, X-Requested-With, Content-Type, Accept, Authorization, Cache-Control');
header('Content-Type: application/json');

// Handle preflight OPTIONS requests
if ($_SERVER['REQUEST_METHOD'] === 'OPTIONS') {
    http_response_code(200);
    exit();
}

// Load Composer autoloader
require_once __DIR__ . '/../vendor/autoload.php';

// Load environment variables
$dotenv = Dotenv\Dotenv::createImmutable(__DIR__ . '/..');
$dotenv->load();

// Initialize application
try {
    // Get route from URL
    $route = $_GET['route'] ?? '';
    $method = $_SERVER['REQUEST_METHOD'];

    // Basic routing system
    $router = new App\Core\Router();
    $router->dispatch($route, $method);

} catch (Exception $e) {
    http_response_code(500);
    echo json_encode([
        'error' => 'Internal server error',
        'message' => $e->getMessage(),
        'timestamp' => date('Y-m-d H:i:s')
    ]);
}

🚀 procédure de démarrage

1. Préparation du projet :
# Clone or create the project directory
git clone [your-repo] atomeek-datamatrix
cd atomeek-datamatrix

# Create the required directory structure
mkdir -p atkm_back/public atkm_back/src/app
mkdir -p database docker/php docker/nginx
mkdir -p atkm/src/app
2. Copier les fichiers de configuration :
  • Copier tous les fichiers de configuration aux emplacements indiqués
  • Ajuster les variables d'environnement dans .env
  • Vérifier les permissions sur les dossiers
3. Démarrage des containers :
# Start all containers
docker-compose up -d

# Check containers status
docker-compose ps

# View logs if needed
docker-compose logs -f [service-name]
4. Installation des dépendances :
# PHP dependencies
docker-compose exec php composer install

# Angular dependencies (if not using containerized Angular)
cd atkm && npm install

8️⃣résolution de problèmes

🐋 problèmes docker

❌ problème : "docker: command not found"

Solution :

  • Redémarrer Docker Desktop
  • Vérifier que Docker Desktop est en cours d'exécution
  • Redémarrer le terminal/PowerShell
  • Réinstaller Docker Desktop si nécessaire

❌ problème : "WSL 2 installation is incomplete"

Solution :

# Update WSL
wsl --update

# Set default version
wsl --set-default-version 2

# Restart Docker Desktop

❌ problème : "Port already in use"

Solution :

# Find process using port
netstat -ano | findstr :8000

# Kill process (replace PID)
taskkill /PID [process_id] /F

# Or change port in docker-compose.yml

🐘 problèmes php

❌ problème : "composer install fails"

Solution :

# Clear composer cache
docker-compose exec php composer clear-cache

# Install with verbose output
docker-compose exec php composer install -vvv

# Update composer
docker-compose exec php composer self-update

❌ problème : "Database connection failed"

Solution :

  • Vérifier que le container MySQL est démarré
  • Vérifier les variables d'environnement dans .env
  • Attendre que MySQL soit complètement initialisé
# Check MySQL logs
docker-compose logs mysql

# Test connection
docker-compose exec php php -r "echo 'MySQL: ' . (new PDO('mysql:host=mysql;dbname=atomeek_datamatrix', 'atomeek_user', 'atomeek_pass') ? 'Connected' : 'Failed');"

🅰️ problèmes angular

❌ problème : "npm install fails"

Solution :

# Clear npm cache
npm cache clean --force

# Delete node_modules and reinstall
rm -rf node_modules package-lock.json
npm install

# Use specific Node version
nvm use 20

❌ problème : "CORS errors in browser"

Solution :

  • Vérifier la configuration proxy dans proxy.conf.json
  • Vérifier les headers CORS dans Nginx et .htaccess
  • Redémarrer les containers après modification

🔧 commandes utiles

# View all container logs
docker-compose logs

# View specific service logs
docker-compose logs -f nginx
docker-compose logs -f php
docker-compose logs -f mysql

# Execute commands in containers
docker-compose exec php bash
docker-compose exec mysql mysql -u root -p

# Restart specific service
docker-compose restart nginx
docker-compose restart php

# Rebuild containers
docker-compose down
docker-compose up --build -d

# Clean Docker system
docker system prune -a
docker volume prune
